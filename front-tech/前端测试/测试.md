测试分类：
+ 静态测试： 代码审查（code review）、静态分析 (typescript)
+ 动态测试： 单元测试、集成测试、端到端测试

测试方案的选择和设计：
1. 单测
	+ 单一功能模块
	+ 测试驱动开发 TDD 模式 
2. 集成测试
	+ 验证模块交互逻辑
	+ 单测之后实施
3. 端到端
	+ 验证最终用户体验
	+ 优先测试*关键路径*（核心功能）

## 单元测试 Unit Test
适用范围： 方法、组件作为测试单元

工具：
+ Vitest √
+ Jest

## 集成测试 Integration Test
适用范围： 多模块之间的交互，涉及到数据库、API 外部依赖

工具：
+ Supertest (NestJs、ReactJs)
+ Cypress

## 端多端测试 End-to-End
适用范围： 站在用户的角度，测试*关键流程*
模拟真实用户的操作

工具：
+ playwright √
+ cypress
