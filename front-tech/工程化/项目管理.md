# Monorepo

>  单仓库多项目

- **定义**：一个仓库里管理多个项目（package/app/library 等）
- **特点**：
    - 所有代码放在同一个仓库中（通常是 `repo`）
    - 项目之间可以共享代码（直接通过本地引用，不用发布到 npm）
    - 通常有统一的依赖管理和构建/发布流程

目录结构：
```js
my-monorepo/
  package.json        <- 根 package.json
  packages/
    app1/
      package.json
    app2/
      package.json
    lib1/
      package.json
```


# pnpm-workspace
- **定义**：pnpm 提供的一种多包管理机制，用于 **在 monorepo 中管理依赖和链接包**
- **特点**：
    - 可以让不同 package 之间通过 **软链接（symlink）** 互相引用，而不需要发布到 npm
    - 自动 hoist（提升）依赖到根目录，避免重复安装
- **工作原理**：
    - 在根 `package.json` 或 `pnpm-workspace.yaml` 中声明哪些目录是 workspace
    - 安装依赖时，pnpm 会把内部包链接起来，同时共享外部依赖

`pnpm-workspace.yaml` 配置
```yaml
packages:
  - "apps/*"
  - "packages/*"
```

`package.json` 配置
```json
{
 "dependencies"： {
 	 "@nihaocafe/shared-api": "workspace:*",
 }
}
```

`vite.config.ts` 配置
```ts
resolve: {
	alias: {
		// 项目地址
		'@nihaocafe/shared-api': path.resolve(__dirname, '../../packages/shared-api/src')
	}
}
```

这样配置后，每个子项目就可以直接访问 pnpm 空间中的导出
`import { something } from 'lib1'`，pnpm 会自动解析到本地 lib 1


Pnpm workspace 是 monorepo 的一个实现方式之一
