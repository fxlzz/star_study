> 那么事件循环呢，是一个很重要的概念，捋一下这个逻辑

首先要搞清楚，JS 这个语言呢是运行在浏览器的一门编程语言。所以我们就要去理解一下浏览器的一些东西

**浏览器是一个多进程多线程的应用程序**

浏览器中有很多的进程，比如说，网络进程，浏览器进程，渲染进程等等

- 网络进程：内部会启动多个线程管理网络资源等
- 浏览器进程：管理其他子进程、用户交互等
- **渲染进程**：会开启一个渲染主线程，在这个线程中，会干很多活，比如解析 HTML、解析 CSS、解析 JS 、一秒画 60 次等

那么**事件循环**呢，会发生在渲染进程上，更准确的话，是在渲染主线程上。

渲染主线程是非常繁忙的，如果没有特定的**调度**算法，它也不知道那个先执行那个后执行。

比如说，
- 全局 JS 执行的好好的，突然来了一个用户点击事件，是停下来执行用户点击事件呢，还是继续执行全局 JS 代码
- 要请求网络资源，JS 代码就卡在那句话等待资源来到吗
- ...

反正有很多问题啊，那么渲染主线程选择了**排队**的一种方式，

上面都是在说问题，那么事件循环就是来解决这些问题的
1. 渲染主线程空闲了，它就会从下面的消息队列（微队列 [ 最高 ]、交互队列 [ 高 ]、延时队列 [ 中 ] 等）中取出任务，然后执行任务
2. 其他线程也可以往消息队列中放入任务（比方说，网络进程中的线程、计时线程等）
3. 一直重复过程

**渲染主线程是永远不会阻塞的！**

## 请你解释事件循环 ？
事件循环是发生在浏览器渲染进程的渲染主线程上的，最重要的三个部件，

1. 执行栈 -> JS 引擎（执行在 JS 线程上 -> 渲染进程上线程）：执行同步代码
2. 任务队列：包含两大部分
	1. 微任务：只有一个微队列
	2. 宏任务：包含有很多队列，比如，延时队列、交互队列等
		在 Chrome 浏览器中，微队列 > 交互队列 > 延时队列
3. 其他线程： 计时线程

## 事件循环的顺序
当执行栈空闲时，会去从任务队列中取任务。任务队列中的任务可能来自同步代码中的一些分任务，也可能来自其他线程的任务。
比如，一个定时器的时间到了（在计时线程中执行），会将它的回调函数包装成一个任务加入到任务队列的尾部。重复上述过程。

**注意：微队列中的任务是要全部清空，才会执行宏任务的（每一次循环只会取一个宏任务，如果浏览器渲染时间还没到，则会在取一个任务）**

## 如何理解 JS 异步？

JS 是一门单线程的语言，运行在浏览器渲染主线程上。

渲染主线程需要做很多事情，比如，解析 HTML、解析 CSS、执行 JS、1 秒钟画 60 次页面等等，如果 JS 没有异步使用同步方案的话，很有可能造成主线程阻塞，消息队列中的其他任务得不到执行。这样一来，不仅会导致繁忙的主线程白白的浪费时间，而且可能导致页面得不到及时的渲染，造成页面卡死。

为了解决这个问题，浏览器就采用了异步的方式来避免。将用户交互、网络请求、计时等功能交给其他的线程去处理。主线程继续执行其他的任务，当其他线程完成后，将事先的回调函数封装成一个任务对象添加到消息队列的末尾，等待主线程的调度。

在这种模式下，浏览器永不阻塞，最大程度上保持了单线程的流畅。


# 一些与 promise 相关的面试题

1. 下面代码的输出结果是什么

```js
const promise = new Promise((resolve, reject) => {
  console.log(1); 
  resolve(); 
  console.log(2);
})

promise.then(() => {
  console.log(3);
})

console.log(4);
```

2. 下面代码的输出结果是什么

```js
const promise = new Promise((resolve, reject) => {
  console.log(1); 
  setTimeout(()=>{
    console.log(2)
    resolve(); 
    console.log(3);
  })
})

promise.then(() => {
  console.log(4);
})

console.log(5);
```

3. 下面代码的输出结果是什么

```js
const promise1 = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve()
  }, 1000)
})
const promise2 = promise1.catch(() => {
  return 2;
})

console.log('promise1', promise1) 
console.log('promise2', promise2) 

setTimeout(() => {
  console.log('promise1', promise1) 
  console.log('promise2', promise2) 
}, 2000)
```

8. 下面代码的输出结果是什么

```js
var a;
var b = new Promise((resolve, reject) => {
  console.log('promise1');
  setTimeout(()=>{
    resolve();
  }, 1000);

  console.log('promise2');
}).then(() => {
  console.log('promise3');

  console.log('promise4');

a = new Promise(async (resolve, reject) => {

  console.log('after1');

console.log('end');
```
