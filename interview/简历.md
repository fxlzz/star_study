## 分析构建和打包结果，优化webpack分包策略，分离非首屏代码，实现懒加载。替换js解析loader，添加webpack缓存策略，将构建时长从30s左右降低至9s

前提：
构建代码 -> 40MB -> 本地开发需 30-32s
打包代码 -> 6.8MB -> 首屏加载时间特别长

从两方面进行优化：
- 构建
- 传输

*构建* 
- js 解析 loader 慢 -> swc-loader
- loader 的解析范围不明确 -> include， 补充 exclude /node_modules/
- 没有缓存 -> 配置 webpack 持久化缓存 cache 
- 采用多进程 -> thread-loader

*传输*
根本原因： webpack optimization 配置项 -> splitChunk 没有配置 -> index.js 这个文件包含所有的依赖项，包特别大
- 进行详细的分包 
	- react 核心依赖包
	- 状态管理库
	- ui 组件库
	- echarts 图表库
	- 自定义库
	- 其他第三方依赖
	- common 库
---
问题一： 分包过多，js 文件达到 124 个
解决：
- 分析那些是首屏的代码，那些不是
	- 分离 -> 进行懒加载 -> 独立的chunk

问题二： 设置懒加载后，并没有被异步加载，而是被 node_modules 捕获了
- A 项目（已经懒加载） ，依赖的 B 项目是静态加载的
	- 解决措施： B 项目是自己团队维护的，直接改源代码 -> 懒加载

## 菜单、权限管理

权限与菜单是绑在一起的，一个角色拥有的权限，其实就是他能看到的菜单。

前端实现`<AuthComponent />`组件，接收一个参数 `key`，描述某个东西的权限key值。
组件只做一件事情，判断这个key是否存在于，后端返回的当前用户的所有key值。如果存在，那么展示；如果不存在，那么不展示。

给一个菜单都写了对应的配置文件 -> menu、func -> 数据处理 -> menu转化成侧边连菜单， func 转化成权限树
